# 日志检索服务介绍

作业帮日志检索服务不对日志进行全文索引，而通过存储压缩的非结构化日志和元数据索引来实现日志的快捷检索。

## 日志检索方法

日志检索服务可按时间窗检索服务日志。服务使用`namespace`/`app`唯一标志，检索时需传入 **`cluster`、`namespace`、`app`、`tp`、`startTime`、`endTime`、`limit`** 等参数；可选参数 **`query`**的写法见下文 **「参数 `query`」**。

## 参数 `tp`：日志类型

- **`tp`**：表示日志属于哪一类，采集落地时写入对应 **log 文件名**；若为空，常见落盘名为 `stdout.log`。

### 按语言栈区分 `tp`（业务 vs 出流量 RAL）

同一套「业务日志 / 出流量 RAL」概念下，**文件名随语言栈不同**；**Mesh 相关 `tp` 与各语言无关**，见下文 Mesh 小节。

**级别习惯（仍适用）**：`warn` / `error` / `fatal` 等常落在带 **`.wf`** 或 **`.wf.log`** 后缀的文件里；常规级别多在对应主文件名上（具体以你们服务实际落盘为准）。

#### Go

业务与框架 RAL 使用 **`server.log` / `module.log` 这一套命名**：

| 组合 | 典型 `tp` / 文件 | 用途概要 |
|------|------------------|----------|
| normal日志 + 业务打印 | `server.log` | 业务打印的常规日志；其中 **notice** 为请求结束由中间件打的访问摘要（uri、耗时、参数/响应等，可选但建议开） |
| 错误日志+ 业务打印 | `server.log.wf` | 业务打印的错误日志 |
| normal日志 + 框架打印 | `module.log` | 框架打印的RAL（出流量）：http/mysql/redis 等出流量；一次成功对外调用常见一条 INFO；监控大盘/链路常依赖其中字段 |
| 错误日志 + 框架打印 | `module.log.wf` | 框架打印的 RAL （出流量）异常相关 |

**RAL（`module.log`）** 通常需关注 `service`、`prot`、`ralCode`、`cost`、`requestStartTime`、`requestEndTime` 等字段。

#### PHP

**出流量 RAL** 不使用 `module.log`，而使用：

| 典型 `tp` / 文件 | 用途概要 |
|------------------|----------|
| `ral-worker.log` | PHP 框架打印的 RAL 出流量常规日志 |
| `ral-worker.wf.log` | PHP 框架打印的 RAL 出流量错误 / 告警侧日志（与 `.wf` 习惯对应） |

（PHP 业务侧其它 `tp` 以服务实际配置为准，此处仅强调与 Go 不同的 RAL 命名。）

### Mesh 流量相关日志（各语言通用）

**与服务语言无关**，所有走 Mesh 的服务均可按方向选用：

| 典型 `tp` / 文件 | 用途概要 |
|------------------|----------|
| `service-mesh.log` | **入 Mesh** 日志：请求进入 Mesh（入站侧） |
| `mesh-out.log` | **出 Mesh** 日志：请求离开 Mesh（出站侧） |

## 参数 `query`：检索表达式
- **`query`**：一段「类 Shell 管道」的检索表达式：使用 `grep` / `grep -E` / `grep -v`等命令进行检索，也支持 `awk`、`wc`、`sort`、`uniq`、`jq` 等命令对结果做进一步的处理，**多个条件用管道符 `|` 连接**，作为**一个字符串**传入日志检索服务，与Shell侧直接grep语法一致。

### 示例

- `grep 4dc6b50dcf824c3c` — 查询包含该字符串的日志
- `grep 4dc6b50dcf824c3c | grep mysql` — 同时包含该字符串与 `mysql`
- `grep 4dc6b50dcf824c3c | grep -v redis` — 包含前者但不包含 `redis`
- `grep -E "cost=[0-9]{4,}." | grep mysql` — 包含 `mysql` 且 `cost` 项在 1s 以上（按正则语义）
- `grep 4dc6b50dcf824c3c | awk -F" " '{print $1}'` — 结果中只保留第一项

### 注意事项

- **查询时间范围**：可选 7 天内任意时间范围；超过 7 天的日志会进入归档。若需查 7 天以前，请在服务中心发起**日志取回申请**，取回完成后再检索。**注意：单次查询的时间窗口不能超过 1 小时**（即 `endTime - startTime <= 3600` 秒），超出会报错。若需查询超过 1 小时的范围，请拆分为多个 ≤ 1 小时的窗口逐次查询。
- **返回条数**：默认最多返回 **200** 条；可在 **Line limit** 中调高，当前最大支持 **10000** 条。
- **单次查询体量**：单次查询日志大小上限约 **1 TB**；超出会提示「日志查询量超出限制，请缩小检索范围后再尝试」，可缩短时间范围或收窄日志类型/检索条件后重试。
